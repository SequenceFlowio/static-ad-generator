# Static Ad Generator
### Claude Code Skill — SequenceFlow

Generate production-ready static ad images for any brand — from brand research to finished
creatives — entirely inside Claude Code. No Higgsfield. No copy-pasting. One command, 5–40 ads.

---

## Prerequisites

- **FAL API key** set as environment variable: `export FAL_KEY="your-key-here"`
- **Python packages**: `pip install requests fal-client`
- **Product images** dropped into the brand folder before running Phase 3

## User Interaction Flow

### Quick start — user says "generate ads for [brand]"
1. Ask for: brand name, brand URL, specific product name
2. Confirm product images are in the folder (or ask them to add images)
3. Run Phase 1 → Phase 2 → Phase 3 sequentially
4. Present the output gallery path

### Selective generation — user says "just run templates 1, 3, 5"
1. Confirm brand folder and prompts.json exist
2. Run Phase 3 with --templates filter

### New product, same brand — user says "now do it for [different SKU]"
1. Skip Phase 1 (Brand DNA already exists)
2. Re-run Phase 2 with new product name
3. Run Phase 3

### Re-research — user says "re-research the brand"
1. Re-run Phase 1 (overwrites brand-dna.md)
2. Re-run Phase 2
3. Run Phase 3

## Phase 1: Brand Research & DNA Generation

**Trigger:** User provides a brand name and URL.

**Instructions for Claude:**

Execute every step in the Brand Research System Prompt below before writing the output.
Save the output as `~/brands/{brand-slug}/brand-dna.md`.

## Phase 2: Prompt Generation

**Trigger:** Brand DNA exists. User wants to generate prompts.

**Instructions for Claude:**
Read `~/brands/{brand-slug}/brand-dna.md`.
Read `~/skills/references/template-prompts.md`.
Fill every template using the rules below.
Save output as `~/brands/{brand-slug}/prompts.json`.

### Prompt Generation Rules

For each template in the library:

1. Replace every [BRACKETED PLACEHOLDER] with brand-specific details from the Brand DNA
2. Prepend the **Image Generation Prompt Modifier** from the Brand DNA to the start of every prompt
3. Set the correct `aspect_ratio`:
   - Feed ads → `"4:5"`
   - Square → `"1:1"`
   - Stories/Reels → `"9:16"`
   - Check the template — it specifies which
4. Set `needs_product_images`:
   - `true` if the template describes the actual product packaging, label, or product in hand
   - `false` if it is pure lifestyle, UGC text card, editorial, or no product shown
5. Keep the `template_number` and `template_name` exactly as defined in the library

### Output JSON Schema

```json
{
  "brand": "Brand Name",
  "product": "Specific Product Name",
  "generated_at": "ISO timestamp",
  "prompts": [
    {
      "template_number": 1,
      "template_name": "headline",
      "prompt": "Full completed prompt text, starting with the Image Generation Prompt Modifier...",
      "aspect_ratio": "4:5",
      "needs_product_images": true,
      "notes": "Any generation notes or copy to refine before running"
    }
  ]
}
```

## Phase 3: Image Generation via FAL API

**Trigger:** prompts.json exists. Product images are in the product-images/ folder.

**Instructions for Claude:**
1. Copy `~/skills/references/generate_ads.py` to `~/brands/{brand-slug}/generate_ads.py`
2. Confirm the user's FAL_KEY is set: `echo $FAL_KEY`
3. Confirm product images exist in `~/brands/{brand-slug}/product-images/`
4. Run the script:

```bash
cd ~/brands/{brand-slug}
python generate_ads.py
```

Or selectively:
```bash
python generate_ads.py --templates 1,3,5
python generate_ads.py --resolution 1K          # faster/cheaper test run
python generate_ads.py --templates 1,2 --resolution 2K
```

### FAL API Details

**Text-to-image endpoint:** `fal-ai/nano-banana-2`
- Input: `prompt`, `aspect_ratio`, `num_images`, `output_format`, `resolution`
- Aspect ratios: `1:1`, `4:5`, `9:16`, `16:9`, `3:2`, `4:3`
- Resolution: `0.5K`, `1K`, `2K`, `4K` — default `2K`

**Image-reference endpoint:** `fal-ai/nano-banana-2/edit`
- Same inputs + `image_urls` (array, up to 14 reference images)
- Product images must be publicly accessible URLs — upload to FAL storage first

## Key Technical Notes

- **1–3 product images is the sweet spot.** Front of packaging, angled shot, lifestyle if available.
- **The /edit endpoint is for product-reference templates only.** When the prompt describes the actual product, use /edit with image_urls so the model sees the real packaging.
- **Aspect ratios matter.** 1:1 for feed, 4:5 for feed (more real estate), 9:16 for stories.
- **Default resolution is 2K** for production quality. Use 1K for fast test runs.
- **4 images per prompt** by default — gives you options to pick the best output.
- **Copy quality:** The prompts from Phase 2 are functional but generic. For higher-converting copy, feed real customer reviews into Phase 2 before running Phase 3.
